# 포도 (PODO) - 안전 경로 안내 서비스

밤길 귀가가 불안한 노약자·여성·아동을 위한 안전 경로 안내 서비스입니다.
빠른 길이 아니라 경찰서, 소방서, 편의점, CCTV 같은 주변 안전시설을 기준으로 점수를 매겨서, 조금 돌아가더라도 안전한 길을 추천해줍니다.

- 기간: 2025.04 ~ 2025.05 (3주)
- 인원: 4인 팀
- 원본 저장소(팀): https://github.com/2025-SMHRD-IS-CLOUD-2/PODO

## 사용 기술

Java, Spring MVC, JSP, MyBatis, MySQL, T-Map API, jQuery

## 담당한 부분

로그인 화면, 메인 지도 + 사이드바, 경로탐색 화면, 보호자 화면 이렇게 4개 화면을 맡았습니다.

- 로그인 화면 만들고 아이디/비번 찾기, 회원가입 이동 처리
- T-Map API 붙여서 지도 화면 만들고, 경찰서/소방서/CCTV/편의점 마커를 카테고리별로 토글해서 켜고 끌 수 있게 함
- 출발지·목적지 입력하면 안전 경로 보여주는 화면 레이아웃
- 보호자 등록/수정/삭제하는 리스트 화면

## 트러블슈팅

**변수명 안 맞아서 데이터가 안 붙던 문제**

백엔드 쪽에서 내려주는 변수명이랑 제가 프론트에서 받으려던 변수명이 서로 달라서 값이 계속 undefined로 찍혔습니다. 처음엔 제 코드가 잘못된 줄 알고 한참 뒤졌는데, 알고보니 이름 자체가 서로 달랐던 거였어요. 백엔드 담당 팀원이랑 얘기해서 이름을 맞추고 나서야 해결됐습니다. 그 뒤로는 API 명세부터 먼저 맞추고 시작하는 습관이 생겼습니다.

**경로탐색 좌표가 안 넘어가던 문제**

출발지 주소를 검색하고 도착지로 넘어가면 지도가 이상한 곳을 가리키는 문제가 있었습니다. 처음엔 AJAX로 좌표를 넘기면 되는 줄 알고 그렇게 짰는데, 페이지가 바뀌면 AJAX로 받았던 값이 그냥 사라져버려서 다음 페이지에서는 좌표가 비어있는 상태였습니다. 콘솔 찍어봐도 계속 null이 나와서 한참 헤맸는데, 개발자도구 네트워크 탭으로 요청을 하나씩 뜯어보다가 원인을 찾았습니다. 결국 좌표를 `addressNavigation.jsp?startLat=...&startLon=...&endLat=...&endLon=...` 처럼 URL에 실어서 넘기는 방식으로 바꿨고, 그제서야 다음 페이지에서도 값이 살아있게 됐습니다.

## 느낀 점

팀 프로젝트를 하면서 코드보다 오히려 협업 쪽에서 배운 게 많았습니다. 특히 변수명 문제는 저 혼자 코드만 계속 들여다봐서는 절대 못 찾았을 문제라, 막히면 일단 팀원한테 물어보는 게 시간을 더 아끼는 길이라는 걸 알게 됐습니다. AJAX 관련해서도 "일단 되는 방식"이 아니라 "왜 이 방식이 맞는지"를 먼저 생각해야 한다는 걸 몸으로 배운 프로젝트였습니다.
