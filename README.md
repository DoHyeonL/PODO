# 포도 (PODO) - 안전 경로 안내 서비스

밤길 귀가가 불안한 노약자, 여성, 아동을 위한 안전 경로 안내 서비스입니다. 최단 경로가 아니라 경찰서, 소방서, 편의점, CCTV 같은 주변 안전시설을 기준으로 점수를 매겨서 조금 돌아가더라도 안전한 길을 추천해줍니다.

- 기간: 2025.04 ~ 2025.05 (3주)
- 인원: 4인 팀
- 팀 원본 저장소: https://github.com/2025-SMHRD-IS-CLOUD-2/PODO

## 사용 기술

Java, Spring MVC, JSP, MyBatis, MySQL, T-Map API, jQuery

## 담당한 부분

로그인, 메인 지도 + 사이드바, 경로탐색, 보호자 등록 이렇게 4개 화면을 맡았습니다.

로그인 쪽은 폼이랑 아이디/비번 찾기, 회원가입 이동까지 만들었고, 메인 화면은 T-Map을 붙여서 지도를 띄우고 경찰서/소방서/CCTV/편의점 마커를 카테고리별로 토글해서 켜고 끌 수 있게 했습니다. 경로탐색 화면은 출발지·목적지를 입력하면 추천 경로를 보여주는 레이아웃이고, 보호자 화면은 등록한 보호자를 수정하거나 지울 수 있게 만들었습니다.

## 트러블슈팅

처음 겪은 건 변수명 문제였습니다. 백엔드에서 내려주는 값이 프론트에서 계속 undefined로 찍혀서 제 코드가 잘못됐다고 생각하고 한참 뒤졌는데, 나중에 보니 애초에 백엔드랑 프론트에서 쓰는 변수명 자체가 달랐습니다. 팀원한테 물어봐서 이름 맞추고 나서야 해결됐고, 이후로는 작업 시작 전에 변수명부터 맞추고 들어갑니다.

그리고 경로탐색에서 출발지 좌표가 도착 페이지까지 안 넘어가는 문제도 있었습니다. 처음에는 AJAX로 좌표를 받아서 바로 다음 화면에 쓰면 될 줄 알았는데, 페이지가 넘어가는 순간 그 값이 그냥 없어져버렸습니다. 콘솔에 계속 null만 찍혀서 한참 헤매다가, 개발자도구 네트워크 탭으로 요청을 하나씩 확인해보고 원인을 알았습니다. 결국 좌표를 AJAX로 들고 있는 대신 페이지를 이동할 때 URL 파라미터에 실어서 같이 넘기는 방식으로 바꿨고, 그렇게 하니 다음 페이지에서도 값을 정상적으로 받을 수 있었습니다.

## 느낀 점

팀플하면서 느낀 건, 막힐 때 코드만 계속 들여다본다고 답이 나오는 게 아니라는 거였습니다. 변수명 문제는 팀원한테 한마디 물어보고 5분 만에 풀렸는데, 그전까지 혼자 몇 시간을 붙잡고 있었거든요. 다음부턴 30분 이상 막히면 일단 물어보자는 나름의 기준이 생겼습니다.
